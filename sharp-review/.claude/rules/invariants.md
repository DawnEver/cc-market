# Sharp Review Invariants

Dev-only constraints on the fan-out scripts (`post-review.js`, `lib.mjs`) — not restating
the agent-facing flow already in `skills/sharp-review/SKILL.md` (and `reference/
task-system.md` / `reference/direct-fanout.md`), which is the runtime source of truth and
must not drift from these.

## Merge / render
- Reviewer fan-out happens in the host (worker subagent or Codex); merge + render run only in
  `post-review.js --raw` via shared `lib.mjs`. Hosts must NOT merge or assign ids themselves —
  that keeps output byte-identical across hosts.

## Schema
- Each reviewer returns `{ findings: [...] }`. A bare array (no `findings` wrapper) causes silent failure.

## Finding IDs
- Format: `SR-YYYYMMDD-NNN`. Assigned sequentially by `mergeFindings` (in `post-review.js --raw`) — never generated by reviewers.
