# Sharp Review Invariants

## Workflow args
- `{ diff, date }` — both required. `date` is YYYY-MM-DD. Never call `Date.now()`/`new Date()` in workflow scripts (breaks resume caching).

## Schema
- StructuredOutput schema must be `{ type: 'object', properties: { findings: [...] } }`. Bare array schema causes silent failure.

## Finding IDs
- Format: `SR-YYYYMMDD-NNN`. Assigned sequentially by the workflow merge phase. Never generated by agents.

## Task sync
- `scripts/sync-tasks.js` is a thin wrapper: parses sharp-review findings → delegates to `rem/scripts/sync-tasks.js`.
- `--resolve SR-ID ...` persists to `.claude/memory/tasks/resolved.txt` (managed by rem engine).
- Checked `[x]` in tasks.md auto-promotes to resolved.txt on next sync.

## Memory cross-reference
- Findings that reference memory entries have their SR-ID written back to the memory file.
- Uses `[[SR-ID]]` wiki-link notation. Enables O(1) lookup on future syncs.
