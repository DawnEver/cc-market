# Sharp Review Invariants

## Workflow args
- `{ diff, date }` — both required. `date` is YYYY-MM-DD. Never call `Date.now()`/`new Date()` in workflow scripts (breaks resume caching).

## Schema
- StructuredOutput schema must be `{ type: 'object', properties: { findings: [...] } }`. Bare array schema causes silent failure.

## Finding IDs
- Format: `SR-YYYYMMDD-NNN`. Assigned sequentially by the workflow merge phase. Never generated by agents.

## Task sync
- `scripts/post-review.js` writes workflow result as a single memory entry (`.claude/memory/YYYY/MM/DD/sharp-review.md`), stamps via `stamp-memory.js`, then directly archives resolved findings to `.claude/tasks/archive/YYYY/MM/DD.md` via `archiveResolved()`.
- Resolution: edit `**Status:** OPEN` → `**Status:** FIXED` directly in the memory file, then `post-review.js --rescan --date YYYY-MM-DD`.
